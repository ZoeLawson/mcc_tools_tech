Documentation life cycle

Before I digress into why I hate Word, a quick overview of what happens when documenting things.

  1. Plan your documentation
  2. Write draft content.
  3. Review content.
  4. Incorporate comments.
  5. Repeat steps 2-4 until content is approved.
  6. Generate final output.
  7. Publish output.

Review content

Nothing you write should go out the door without someone else looking at it.

  • Technical review - SMEs, Quality Assurance (QA) (Testers), and Implementers

    Is the content correct?

  • Copy edit - Review for grammar, spelling, follow company style, etc.

    Some of this may be automated, but a human is best.

Incorporate comments

Once you get review comments, you'll need to make revisions.

This can sometimes get exciting when reviewers have differences of opinion.

Depending upon the formality of your review process, you may need to track the resolution of the comment.

Repeat

Often, after incorporating comments, you need to put things out for review again.

The review process is very iterative.

You may need to perform reviews in a certain order, e.g. technical review before implementer review.

You should have some sort of approval/sign-off process.

Even if it's just an email thread (CYA)

Generate final output

What you publish should be more refined than your review drafts.

Most likely several types of output, at least PDFs and HTML.

May need different versions of the same content

There should be a checklist, may be considered a separate review process. e.g. remove all comments, confirm copyright date, TOC/bookmarks generate properly, links work, graphics correct quality.

One of my tech writing priorities

To understand my loathing of using Office products for documentation, understand my priorities.

Documentation should be:

  • Sustainable and Maintainable
  • Repeatable and Automatable

Can I easily maintain multiple versions?

Can I easily maintain similar but different products/components?

Can I get all the output I need easily?

Templates and Styles

They almost work...

A Word Template (.dot or .dotx) is a document that contains formatting options.

You need to keep them in C:\Users\<YourAccount>\Documents\Custom Office Templates

Often easier to create a new document based on a new template vs applying an existing template

(Applying an existing template assumes you are using the same style names)

Create new document

Make sure that your .dotx files are in your Custom Office Templates folder. (C:\Users\<YourUserName>\Documents\Custom Office Templates)

  1. Select File > New.

    Using CTRL+N doesn't get you to the template option.

  2. Select Personal next to Featured.

    Don't follow the Suggested Search for Personal.

  3. Select your template.

This creates a copy of template as a Word document. This should include boilerplate text and images.

Middlesex Community College - Tools and Technologies for Tech Writers - Winter 2024

Importing a template

This is for Word 2013, YMMV

  1. Select File > Options.
  2. Select Add-ins.
  3. Set Manage to Templates and click Go.
  4. Click Attach and find the .dotx or .dot file.
  5. Click OK.

You then need to re-apply styles as necessary. If you do this to a blank document, you will need to recreate the boilerplate content.

Using Styles

Stop using the quick formatting tools

The styles section of the ribbon or the panel are fine.

When formatting paragraphs, use paragraph styles with the Â¶ mark.

For inline formating, use character styles with the a mark.

If you need to change formatting, go to the Style, right-click and select Modify.

Fixing Styles

Beware of Style Name + Formatting

The important thing about using Styles is that you should never, ever use any of the quick formatting options with them.
